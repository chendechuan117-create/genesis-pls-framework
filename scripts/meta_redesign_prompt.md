# Genesis 元信息重构：聚焦 Prompt

以下 prompt 用于给 Genesis 发送（通过 Discord 对话或 /auto 的自定义任务），让它基于已有的知识节点，逐步推演元信息的新架构。

---

## 方式一：直接对话（推荐先试这个）

发送以下内容给 Genesis：

```
/deep 搜索知识库中所有标签含 "llm_physics" 的 LESSON 节点。这些是从外部观测 LLM 行为推导出的 6 条物理定律 + 3 条推论。

你的任务不是评价这些定律对不对，而是：

1. 先逐条读完所有 llm_physics 节点，理解它们的推导链
2. 然后回答一个具体的设计问题：

【设计问题】
从这 6 条定律出发，推导出元信息系统的最小可行架构。要求：
- 每一步推导必须引用具体的定律编号作为依据
- 不允许引入定律之外的假设（尤其是来自人类认知的假设）
- 如果推导到某一步发现定律不够用，明确指出"此处缺少定律"而不是自行补充

最终输出格式：
- 推导过程（每步标注依据的定律）
- 最小架构描述（不超过 500 字）
- 该架构与 Genesis 当前系统的具体差异点（列表）
- 推导中发现的"定律缺口"（如果有）

注意：你的知识库中有一个节点 LESSON_CURRENT_METADATA_CHIMERA 描述了当前系统的问题。另一个节点 LESSON_METADATA_FIELD_MODEL 描述了一个候选方向。请在推导过程中验证这个候选方向是否能从定律严格推导出来，还是包含了额外假设。
```

## 方式二：修改 auto 首轮 prompt（如果想用 /auto 跑多轮）

替换 `AUTO_PROMPT_FIRST` 为以下内容（不改代码结构，只改文本）：

```python
AUTO_PROMPT_FIRST = """你正在执行一个特殊的自主推演任务：从 LLM 物理定律推导元信息系统的最小可行架构。

## 背景
你的知识库中有一组标签为 llm_physics 的 LESSON 节点（约 10 个），它们是从外部观测 LLM 行为推导出的定律和推论。这些不是假说，而是观测事实。

## 本轮任务
1. 搜索并读取所有 llm_physics 标签的知识节点
2. 从这些定律出发，逐步推导元信息系统的设计约束
3. 每一步推导必须引用具体的定律作为依据，不允许引入额外假设
4. 如果推导到某步发现定律不够用，记录为"定律缺口"而不是自行补充
5. 将推导结论写入知识库（调用 record_lesson_node），标签包含 meta_redesign_derivation

## 规则
- 不要修改任何代码
- 不要使用 Doctor 沙箱
- 纯知识推演任务：搜索 → 推导 → 记录结论
- 每轮只推导一步，做到位，不要跳步

## 输出要求
- 明确写出"本轮推导步骤"和"推导依据（定律编号）"
- 如果发现已有节点之间存在矛盾或缺口，记录为新的 LESSON 节点

以下是当前系统信号（仅供参考，本轮不需要处理系统问题）：
{signals}"""
```

对应的 `AUTO_PROMPT_CONTINUE` 改为：

```python
AUTO_PROMPT_CONTINUE = """继续元信息架构推演。不要重复上一轮已完成的工作。

上一轮工作记忆：
{knowledge_state}

上一轮推演前沿：
{frontier_state}

{history}

## 续跑原则
- 沿上一轮推导的前沿继续下一步推导
- 每步必须引用 llm_physics 定律作为依据
- 如果上一轮推导有逻辑跳跃，先修补再继续
- 将本轮推导结论写入知识库，标签包含 meta_redesign_derivation

## 推演方向参考
以下是整体推演路线图（不要求一次全部完成，每轮只推一步）：
1. 从定律推导出元信息的存在形式约束
2. 从约束推导出最小架构
3. 从最小架构推导出场模型的具体设计
4. 设计场的演化机制（基于定律6：观测>反思）
5. 设计场和深层知识的接口
6. 与当前 Genesis 架构的差异分析和迁移路径

当前信号（仅供参考）：
{signals}"""
```

## 审视要点（我来做的部分）

Genesis 产出后，我需要检查：
1. **是否在套壳** —— 推导中是否偷偷引入了定律之外的假设（尤其是人类认知假设）
2. **是否有实质推进** —— 每轮是否产生了新的、不可由上一轮直接得出的结论
3. **定律缺口是否真实** —— 报告的"定律不够用"是真的需要新观测，还是推导不到位
4. **知识节点质量** —— 记录的 LESSON 节点是否精确、可复用、无冗余
